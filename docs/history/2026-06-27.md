## やったこと

### Phase 2-7: XGBoost をアンサンブルに追加

**背景:**
- romantamrazov v2 の Ridge Stack で LGB2 の weight が 0.02（ほぼ死んでいる）
- LGB と XGBoost は木の構築方針が異なる（leaf-wise vs depth-wise）→ 多様性期待
- Kriging 路線を棚上げして低リスクな手を打つ

**構成変更:** LGB0 + LGB1 + ~~LGB2~~ → **XGBoost** + CB

#### v1 失敗: `tree_method='gpu_hist'` エラー

```
XGBoostError: Invalid Input: 'gpu_hist', valid values are: {'approx', 'auto', 'exact', 'hist'}
```

XGBoost 2.0+ で `gpu_hist` が廃止。GPU 使用は `device='cuda'` で指定。

**修正:** `tree_method='gpu_hist'` → `tree_method='hist'` + `device='cuda'`（変わらず）

#### v2 失敗: `early_stopping_rounds` の位置エラー

```
TypeError: XGBModel.fit() got an unexpected keyword argument 'early_stopping_rounds'
```

XGBoost 2.0+ では `early_stopping_rounds` は `fit()` でなくコンストラクタに渡す。
（v2 で誤って `fit()` に移動してしまったため）

**修正:** `fit()` から `early_stopping_rounds=250` を削除（`XGB_PARAMS` に残す）

#### v3 結果: **LB 10.342**（romantamrazov v2 比 +0.058 改善）

```python
XGB_PARAMS = dict(
    n_estimators=8000,
    learning_rate=0.025,
    max_depth=7,
    min_child_weight=15,
    subsample=0.75,
    colsample_bytree=0.75,
    reg_lambda=3.0,
    reg_alpha=0.05,
    tree_method='hist',   # XGBoost 2.0+ 形式
    device='cuda',
    objective='reg:squarederror',
    random_state=42,
    early_stopping_rounds=250,  # コンストラクタに渡す
    verbosity=0,
)
```

## 技術メモ

### XGBoost 2.0+ の GPU 設定変更まとめ

| 項目 | 旧（<2.0） | 新（2.0+） |
|---|---|---|
| GPU tree method | `tree_method='gpu_hist'` | `tree_method='hist'` |
| GPU 指定 | `tree_method='gpu_hist'` で兼用 | `device='cuda'` を別途指定 |
| early stopping | `fit(early_stopping_rounds=N)` | `XGBRegressor(early_stopping_rounds=N)` |

## Phase 2-8: IRLS 多項式平滑化（同日 push）

公開上位解法（Physics Ensemble, dual-pipeline）で共通する後処理を追加。

**手法:** per-well で `u = TVT + Z`（構造標高）を degree-4 多項式 + IRLS（4回反復）でフィット。
- IRLS 重み: Cauchy `w = 1/(1 + (r/2σ)²)`
- ブレンド: `0.25 × raw + 0.75 × poly_fit`
- anchor = last_known_TVT + Z（既知プレフィックスの末尾を基準に相対化）
