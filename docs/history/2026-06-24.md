## 分析・考察

### Kriging/GP が IDW に勝てない理由

centroid ベースの近傍選択が根本問題。

| 手法 | 近傍選択 | eval 点ごとの適応 |
|---|---|---|
| IDW（DenseANCCImputer）| **per-row** に20近傍 | ✅ 各点で最適 |
| numpy Kriging（v2） | centroid 基準150点 | ❌ 末端で不適切 |
| sklearn GP（v3） | centroid 基準80点 | ❌ 末端で不適切 |

長い水平ウェル（eval ゾーンが2000ft以上）では centroid から遠い末端の eval 点が
本来の最近傍訓練データを使えないため、IDW の per-row 選択に劣る。

### 3日間の Kriging 実験まとめ

| 手法 | OOF |
|---|---|
| IDW（romantamrazov v2） | 10.27 |
| numpy Kriging（Range 過小推定） | 11.76 |
| sklearn GP Matérn（centroid 近傍） | 10.92 |

→ **Kriging 路線は一旦棚上げ**。IDW の per-row 近傍を活かした別アプローチへ。

## 次の方針

**XGBoost 追加**（romantamrazov v2 の LGB2 を置換）

- LGB2 の Ridge 重みが 0.02〜0.16 とほぼ死んでいる
- LGB と XGBoost は木の構築方針が異なる（leaf-wise vs depth-wise）ため多様性が出る
- 実装コスト低、リスク低

**期待改善:** -0.2〜0.3（OOF 10.27 → 10.0 前後）
